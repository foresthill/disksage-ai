## 1. エグゼクティブサマリー

DiskSage は、macOS / Windows の個人ユーザー向け容量管理アプリ。既存の CleanMyMac や DaisyDisk と異なり、ファイルの「正体」を AI（Claude API）が判定し、削除の可否をユーザーに提案する。自動削除は一切行わない——すべてのアクションは人間が承認する。

コアコンセプトは3つ：

- **提案方式（Never auto-delete）**: 安全第一、ユーザーが最終判断
- **AI 判定**: サイズや拡張子ではなく「中身」を読んで判定（壊れたバックアップ、孤児データ、古いキャッシュを識別）
- **OSS**: コア機能は Apache 2.0、信頼性とコミュニティによる成長

## 2. 背景：実話ベースの課題

本企画は、開発者 Foresthill が自身の Mac（460GB SSD）で体験した次のような問題群から生まれた。

### 2.1 解決すべき現実の課題

1. 60GB 解放しても数週間で空き容量ゼロに戻る（増殖の根本原因が見えない）
2. Zoom 会議中にメール受信ができなくなる（macOS が書き込み制限に入る二次被害）
3. 半年前に入れたアプリ iMobie が 39GB のデータを残して孤児化（しかもバックアップは破損）
4. 開発者特有の node_modules、Docker.raw、APFS スナップショット、Whisper モデルなどが無自覚に肥大
5. 既存ツールはサイズ順に並べるだけで、「これは何？」「消していい？」の判断を人間に丸投げ

### 2.2 市場のギャップ

| ツール | 方式 | 課題 | 価格 |
|---|---|---|---|
| CleanMyMac X | ルールベース自動削除 | クロ箱、プライバシー懸念、年額高い | ¥5,900/年〜 |
| DaisyDisk | サイズ可視化 | 判定は人間まかせ、洞察なし | ¥1,500 買切 |
| GrandPerspective / OmniDiskSweeper | サイズ可視化（OSS/無料） | UI が開発者向け、判定支援ゼロ | 無料 |
| WinDirStat (Win) | サイズ可視化 | 2000年代の UI、放置気味 | 無料 |
| **DiskSage（本企画）** | **AI 判定 × 提案** | — | **OSS + Pro $5/月** |

## 3. ソリューション

DiskSage は 4 つの層で動作する。

### 3.1 スキャンエンジン（Rust コア）

- OS 固有の典型的肥大ポイントを既知パターンでスキャン（Docker.raw、node_modules、APFS スナップショット、Library/Application Support 配下など）
- ファイルシステム全体のインベントリを構築（ホーム配下のみ、権限付与で全体も可）
- パターンライブラリは OSS で成長——ユーザーが匿名で共有（オプトイン）

### 3.2 AI 判定レイヤー（Claude API）

- ファイル名、ディレクトリ構造、Info.plist / manifest / 拡張子を LLM に渡して「正体」を推定
- 破損判定：iPhone バックアップなら Manifest.db の有無、npm なら package.json との整合性、など
- 使用頻度判定：最終アクセス時刻 + ディレクトリ意味論で「実質未使用」を識別
- BYOK（Bring Your Own Key）対応：ユーザー自身の Claude API キーで動く。ローカル LLM（Ollama）にもフォールバック可能

### 3.3 提案 UI

- Tinder 式のカード UI：1 件ずつ「残す / 外付けに退避 / ゴミ箱へ / 今は保留」を選択
- 各提案に AI の推論ロジックを明示（なぜ削除候補と判定したか）
- 一括承認モード：同カテゴリ（例: node_modules）をまとめて承認
- 削除は常にゴミ箱経由。完全削除は別ステップで明示的に

### 3.4 継続監視

- バックグラウンドで空き容量トレンドをグラフ化
- 閾値（例: 20GB 未満）を割ったら通知
- 週次の自動スキャン提案（削除は行わず、レポートのみ）

## 5. MVP（Phase 1）スコープ

最初の 6 週間で出す版。macOS 限定、一次ターゲットに絞る。

- 継続監視・通知（Phase 2）
- 匿名パターン共有（Phase 3）

- 外付けドライブ退避ウィザード（Phase 2）

## 7. 技術アーキテクチャ

### 7.1 採用スタック

| レイヤー | 採用 | 理由 |
|---|---|---|
| デスクトップシェル | Tauri 2.x (Rust + WebView) | Electron より軽量。容量管理アプリ自体が重い自己矛盾を回避 |
| UI | React + Tailwind + shadcn/ui | 開発速度、コミュニティ大 |
| スキャンエンジン | Rust（walkdir, tokio） | 数百万ファイルを扱う前提、パフォーマンス必須 |
| AI 統合 | Anthropic SDK（TypeScript） | Claude API ネイティブ連携、BYOK |
| データ永続化 | SQLite（via sqlx） | ローカル完結、外部依存ゼロ |
| アップデート配信 | Tauri updater + GitHub Releases | OSS 標準、署名済み配布 |

### 7.2 セキュリティとプライバシー

- API 経由で LLM に送るのは「ファイル名・ディレクトリ構造・メタデータ」のみ。ファイルの中身は一切送信しない
- ユーザーの API キーは Keychain（Mac）/ Credential Manager（Win）に保存、平文保存しない
- 削除は常にゴミ箱経由、完全削除は追加の明示承認
- OSS なので監査可能、透明性担保

## 9. 競合優位性（Moat）

1. **AI 判定の正確性**：Claude の推論力 × ドッグフーディングで磨かれたパターンライブラリ
2. **「勝手に消さない」という思想的な差別化**：自動削除アプリへの不信感が市場に存在
3. **OSS による信頼**：プライバシーを気にするユーザーは商用クロ箱を避ける
4. **開発者ファースト**：MVP ターゲットが開発者なので、Dogfooding と UX 改善サイクルが高速
5. **Anthropic エコシステム**：Claude Apps としての展開、Claude Code との連携可能性

| リスク | 影響度 | 対策 |

## 12. 付録：Dogfooding ケーススタディ（2026-04-20）

本企画の出発点となった、Foresthill の Mac で実際に発見された肥大要因の一覧。これらすべてが DiskSage の検出パターンのシードとなる。

| 項目 | サイズ | 性質 | DiskSage での判定例 |
|---|---|---|---|
| iMobie iPhone backup（孤児） | 39 GB | 破損バックアップ | Manifest.db 不在 → 復元不可、削除推奨 |
| Claude Desktop キャッシュ | 9.5 GB | 再生成可能キャッシュ | Cache/GPUCache 等は安全削除、Local Storage は残す |
| VoiceInk モデル + 録音履歴 | 8.1 GB | LLM モデル | 未使用 Whisper モデル識別、古い録音提案 |
| Ollama モデル | 11 GB | LLM モデル | 最終使用日時ベースで未使用モデル検出 |
| Development 配下 node_modules | 10+ GB 見込 | ビルド副産物 | 30 日以上未編集のプロジェクトを対象に提案 |
| Library/Caches 全般 | 24 GB | アプリキャッシュ | アプリ別に安全度を提示、Dropbox や Homebrew は高安全 |
| Library/Developer（Xcode） | 14 GB | 開発者ツール副産物 | DerivedData, Archives, Simulator を個別判定 |
| macOS VM swap | 22.6 GB | スワップファイル | 再起動推奨サイン |
| /Library/Application Support | 46 GB | システム共有アプリデータ | ホーム外の見えない占有、個別判定要 |
